# antigravity.google 官网粒子完整逆向（精读版）

## 一、粒子类 `rI`（压缩名）= `K9`

### 工厂方法 `static async create(scene, textures)`
```js
static async create(e, t) {
  let i = new rI(e, t);
  i.createPoints();              // 1. 基础泊松盘采样（固定密度）
  await i.createPointsFromImage(); // 2. 加载图片 + Worker 算最近邻
  i.init();                      // 3. 建 GPGPU 纹理/RT/shader/mesh
  return i;
}
```

### `createPoints()` — 基础分布（固定密度泊松盘）
```js
let t = new PoissonDiskSampling({
  shape: [500, 500],
  minDistance: linearMap(density, 0, 300, 10, 2),   // density=100 → 6.67
  maxDistance: linearMap(density, 0, 300, 11, 3),   // density=100 → 7.0
  tries: 20
}).fill();
this.pointsBaseData = t;
this.pointsData = [];
for (let i = 0; i < t.length; i++)
  this.pointsData.push(t[i][0] - 250, t[i][1] - 250);  // 中心化到 [-250,250]
this.count = this.pointsData.length / 2;
```
**注意：这是纯随机均匀泊松盘，不是水母环形！粒子形状完全由图片决定。**
`minDistance`/`maxDistance` 接近相等（density=100 时 6.67 vs 7.0）→ 近似均匀网格。

### `createPointsFromImage()` — 图片驱动的最近邻
```js
// 1. 加载每张图片(默认 icon_cube.png) → imageData 500×500
// 2. 对每张图片，用 Worker 算 pointsBase 每个点的"最近邻"(基于图片亮度)
//    nearestPointsData[textureIndex] = [x1,y1, x2,y2, ...] (中心化 -250)
```

### Worker 内部逻辑（关键！）
```js
const distanceFunction = (point, imageData) => {
  const idx = (Math.round(point[0]) + Math.round(point[1]) * imageData.width) * 4;
  const pixel = imageData.data[idx] / 255;
  return pixel * pixel * pixel;   // 亮度的立方 → 暗处距离更近(更密)
}
const maxDistance = linearMap(density, 0, 300, 10, 50);
const poissonDisk = new PoissonDiskSampling({
  shape: [500, 500], minDistance: 1, maxDistance: maxDistance, tries: 20,
  distanceFunction: point => distanceFunction(point, imageData)  // 变密度
});
const points = poissonDisk.fill();   // 图片驱动的粒子分布

// 对 pointsBase 每个点，在 points 里找"亮度<1 且距离最近"的点
for (let i = 0; i < pointsBase.length; i++) {
  let nearestPoint = null, nearestDistance = Infinity;
  for (let j = 0; j < points.length; j++) {
    if (Math.random() < 0.75) continue;   // 75% 跳过(性能优化，随机采样)
    const distance = Math.hypot(points[j][0]-pointsBase[i][0], points[j][1]-pointsBase[i][1]);
    const pixelRedValue = distanceFunction(points[j], imageData);
    if (pixelRedValue < 1 && distance < nearestDistance) {
      nearestDistance = distance;
      nearestPoint = points[j];
    }
  }
  nearestPoints.push(nearestPoint[0] - 250, nearestPoint[1] - 250);
}
```
**核心：粒子有两个位置集**
- `pointsData`（posTex/uPosRefs）：均匀网格基础位置（粒子"家"）
- `nearestPointsData`（uPosNearest）：图片轮廓上的点（hover 时粒子被"吸"过去勾勒图片轮廓！）

### `createDataTexturePosition(data)` — RGBA = [x,y,scale,velocity]
```js
let t = new Float32Array(this.length * 4);  // length = 256*256 = 65536
for (let r = 0; r < this.count; r++) {
  t[r*4+0] = data[r*2+0] * (1/250);   // 归一化到 [-1,1]
  t[r*4+1] = data[r*2+1] * (1/250);
  t[r*4+2] = 0;   // scale
  t[r*4+3] = 0;   // velocity
}
new DataTexture(t, 256, 256, RGBAFormat, HalfFloatType);
```
**官网用 HalfFloatType！但 Float32Array 配 HalfFloat 会报错——官网能跑是因为它 HalfFloat 打包用 Uint16。我们用 FloatType 配 Float32Array 最稳。**

## 二、SIM shader（逐像素物理）— 完整原文
```glsl
uniform sampler2D uPosition, uPosRefs, uPosNearest;
uniform vec2 uMousePos;
uniform float uTime, uDeltaTime, uIsHovering;
// uRingRadius/uRingWidth/uRingWidth2/uRingDisplacement 声明了但 shader 内未使用！

vec2 hash(vec2 p){
  p = vec2(dot(p,vec2(2127.1,81.17)), dot(p,vec2(1269.5,283.37)));
  return fract(sin(p)*43758.5453);
}

void main() {
  vec2 uv = gl_FragCoord.xy / 256.0;
  vec4 pFrame = texture2D(uPosition, uv);
  float scale = pFrame.z;
  float velocity = pFrame.w;
  vec2 refPos = texture2D(uPosRefs, uv).xy;       // = 初始位置
  vec2 nearestPos = texture2D(uPosNearest, uv).xy; // 图片轮廓点
  float seed = hash(uv).x, seed2 = hash(uv).y;

  float time = uTime * 0.5;
  float lifeEnd = 3.0 + sin(seed2*100.0)*1.0;      // 2~4s 生命周期
  float lifeTime = mod((seed*100.0) + time, lifeEnd);

  vec2 pos = pFrame.xy;
  float distRadius = 0.15;

  // hover 时目标从 refPos 渐变到 nearestPos（勾勒图片轮廓！）
  vec2 targetPos = mix(refPos, nearestPos, uIsHovering * uIsHovering);
  vec2 direction = normalize(targetPos - pos) * 0.01;
  float dist = length(targetPos - pos);
  float distStrength = smoothstep(distRadius, 0.0, dist);  // 越近力越大
  if (dist > 0.005) pos += direction * distStrength;

  // 生命重置
  if (lifeTime < 0.01) { pos = refPos; pFrame.xy = refPos; scale = 0.0; }

  // scale 渐入渐出 + hover 放大
  float targetScale = smoothstep(0.01,0.5,lifeTime) - smoothstep(0.5,1.0,lifeTime/lifeEnd);
  targetScale += smoothstep(0.1,0.0,smoothstep(0.001,0.1,dist)) * 1.5 * uIsHovering;
  scale += (targetScale - scale) * 0.1;

  // velocity = 接近程度 * hover
  velocity = smoothstep(distRadius, 0.001, dist) * uIsHovering;

  vec2 finalPos = pos;   // disp=0，所以 finalPos=pos
  vec2 diff = finalPos - pFrame.xy;
  diff *= 0.2;
  gl_FragColor = vec4(pFrame.xy + diff, scale, velocity);
}
```

## 三、RENDER vertex shader — 完整原文
```glsl
attribute vec4 seeds;
uniform sampler2D uPosition;
uniform float uTime, uParticleScale, uPixelRatio, uIsHovering, uPulseProgress;
uniform int uColorScheme;
varying vec4 vSeeds; varying float vVelocity;
varying vec2 vLocalPos, vScreenPos; varying float vScale;
${noise}

void main() {
  vec4 pos = texture2D(uPosition, uv);
  vSeeds = seeds;

  // 双层 simplex 噪声
  float noiseX = snoise(vec3(pos.xy*10.0, uTime*0.2+100.0));
  float noiseY = snoise(vec3(pos.xy*10.0, uTime*0.2));
  float noiseX2 = snoise(vec3(pos.xy*0.5, uTime*0.15+45.0));
  float noiseY2 = snoise(vec3(pos.xy*0.5, uTime*0.15+87.0));

  // 径向脉冲波（pushProgress 从中心向外）
  float cDist = length(pos.xy);
  float progress = uPulseProgress;
  float t = smoothstep(progress-0.25, progress, cDist) - smoothstep(progress, progress+0.25, cDist);
  t *= smoothstep(1.0, 0.0, cDist);
  pos.xy *= 1.0 + (t * 0.02);

  // hover 时近鼠标粒子抖动
  float dist = smoothstep(0.0, 0.9, pos.w);
  dist = mix(0.0, dist, uIsHovering);
  pos.y += noiseY * 0.005 * dist;
  pos.x += noiseX * 0.005 * dist;
  pos.y += noiseY2 * 0.02;    // 全局漂浮
  pos.x += noiseX2 * 0.02;

  vVelocity = pos.w; vScale = pos.z; vLocalPos = pos.xy;
  vec4 viewSpace = modelViewMatrix * vec4(vec3(pos.xy,0.0), 1.0);
  gl_Position = projectionMatrix * viewSpace;
  vScreenPos = gl_Position.xy;

  float minScale = 0.25 + float(uColorScheme) * 0.75;
  gl_PointSize = ((vScale*7.0) * (uPixelRatio*0.5) * uParticleScale) + (minScale * uPixelRatio);
}
```

## 四、RENDER fragment shader — 完整原文

```glsl
varying float vScale, vVelocity;
uniform vec3 uColor1, uColor2, uColor3;
uniform vec2 uMousePos, uRez;
uniform float uAlpha, uTime;

${noise}

float sdRoundBox(vec2 p, vec2 b, vec4 r){...}
vec2 rotate(vec2 v, float a){...}

void main() {
  float uBorderSize = 0.2;
  float ratio = uRez.x / uRez.y;
  float angle = atan(vLocalPos.y-uMousePos.y, vLocalPos.x-uMousePos.x); // 算了但没用！

  vec2 uv = gl_PointCoord.xy - 0.5;
  uv.y *= -1.0;

  vec2 tuv = vScreenPos;

  tuv.y *= 1.0/ratio;
  tuv += 0.5;

  // 颜色：velocity 三段渐变
  float h = 0.8;
  float progress = vVelocity;

  float dist = length(uv);
  float dr = 0.5;
  float t = smoothstep(dr+(uBorderSize+0.0001), dr-uBorderSize, dist);  // 算了没用

  float rounded = sdRoundBox(uv, vec2(0.5,0.2), vec4(0.25));  // 圆角矩形
  rounded = smoothstep(0.1, 0.0, rounded);                      // 算了没用！

  float disc = smoothstep(0.5, 0.45, length(uv));  // ★ 实际 alpha 用 disc

  float a = uAlpha * disc * smoothstep(0.1, 0.2, vScale);
  if (a < 0.01) discard;

  color = clamp(color, 0.0, 1.0);
  color = mix(color, color*clamp(vVelocity,0.0,1.0), float(uColorScheme));
  gl_FragColor = vec4(color, clamp(a, 0.0, 1.0));
  #ifdef SRGB_TRANSFER
    gl_FragColor = sRGBTransferOETF(gl_FragColor);
  #endif
}
```

## 六、与我的实现的关键差异（导致效果不对的原因）

| 项 | 官网 | 我的实现 | 影响 |
|----|------|---------|------|
| **粒子分布** | 图片驱动变密度泊松盘（icon_cube.png 轮廓） | 三层手工圆环 | 形态完全不同 |
| **uPosNearest** | 图片轮廓点（hover 时粒子吸过去勾勒图标） | 随机偏移点 | hover 行为不同 |
| **mousePos 范围** | intersectionPoint×0.175（world space，z=0平面） | NDC×0.175 | 鼠标映射错位 |
| **camera** | PerspectiveCamera(40°) zoom=3.5，raycaster 打平面 | PerspectiveCamera(40°) 无 raycaster | 透视不同 |
| **mesh.scale** | (5,-5,5) | (5,-5,5) ✓ | OK |
| **hoverProgress 缓动** | GSAP power3.out 0.5s | 指数 lerp | 节奏不同 |
| **pushProgress** | GSAP 0→1 2s power2.out，hover start/end 各触发 | 自写 easeOutQuad | 节奏不同 |
| **uColorScheme** | dark=0,light=1，影响 minScale 和颜色 | 未实现 | 暗色模式基础尺寸小 |
| **FloatType** | HalfFloatType(+Uint16打包) | FloatType ✓ | OK(等价) |
| **粒子形状** | disc(圆点)，sdRoundBox 算了没用 | capsule(我改造) | 这是你要求的改造 |
| **pulseProgress** | pushProgress 直接传入(0→1线性时间) | 同 ✓ | OK |

## 七、结论：要做到像官网
1. **粒子分布用图片驱动**（这是官网粒子形状的来源，不是水母环）
2. **uPosNearest = 图片轮廓点**（hover 勾勒图标）
3. **mousePos 用 raycaster**（透视正确）
4. **GSAP 缓动 hover/push**（节奏对）
5. **uColorScheme=0**（dark 模式 minScale=0.25）
6. 粒子形状：你要求改成 capsule（sdRoundBox 改长条 + 用作 alpha）
